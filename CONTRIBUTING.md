# contributing

## commits

- format: `feat:`, `fix:`, `chore:`, `docs:`, … then **lowercase** description
- prefer **one commit per small change** (single file, single handler, single route)
- target **hundreds of granular commits** over time rather than few large ones

## scope

- avoid editing `sdks/zep-python` unless you are intentionally upgrading the sdk
- prefer changes in `backend/`, `frontend/`, `examples/`, and compose/docs
